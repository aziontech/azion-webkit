# Button

Botões são elementos interativos fundamentais que permitem aos usuários realizar ações e interagir com a interface.

## Visão Geral

O componente Button fornece uma maneira consistente e acessível de acionar ações em sua aplicação. Ele suporta múltiplas variantes, tamanhos e estados para cobrir diversos casos de uso.

<ExampleBlock title="Exemplo Básico">
  <AzButton variant="primary">Botão Primário</AzButton>
</ExampleBlock>

## Variantes

### Primary

Use para ações principais e CTAs (Call to Action).

<ExampleBlock title="Primary Button">
  <AzButton variant="primary">Primário</AzButton>
</ExampleBlock>

### Secondary

Use para ações secundárias ou alternativas.

<ExampleBlock title="Secondary Button">
  <AzButton variant="secondary">Secundário</AzButton>
</ExampleBlock>

### Destructive

Use para ações destrutivas ou perigosas, como excluir.

<ExampleBlock title="Destructive Button">
  <AzButton variant="destructive">Excluir</AzButton>
</ExampleBlock>

### Ghost

Use para ações sutis ou terciárias.

<ExampleBlock title="Ghost Button">
  <AzButton variant="ghost">Fantasma</AzButton>
</ExampleBlock>

## Acessibilidade

### Navegação por Teclado

| Tecla | Ação |
|-------|------|
| Tab | Move o foco para o botão |
| Enter | Ativa o botão quando focado |
| Space | Ativa o botão quando focado |

### ARIA

- Use `aria-disabled="true"` em vez do atributo `disabled` para leitores de tela
- Use `aria-label` para botões apenas com ícone
- Use `aria-busy="true"` quando o botão estiver carregando
