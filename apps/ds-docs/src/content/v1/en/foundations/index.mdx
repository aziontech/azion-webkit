# Foundations

Foundations define the visual and interaction principles that guide every component and interface across the design system.

They ensure consistency, accessibility, and scalability across all products and experiences.

## What are Foundations

Foundations are the core design decisions that underpin the entire system. They define the fundamental rules that ensure visual and functional coherence.

Every component in the design system is built on top of these foundations. Understanding them helps you:

- Make informed design decisions
- Create consistent interfaces
- Maintain scalability across products
- Ensure accessibility compliance

Foundations define:

- **Color systems** — Palettes, semantic colors, and accessibility guidelines
- **Typography scale** — Font families, sizes, weights, and line heights
- **Spacing rules** — Consistent margins, padding, and gaps
- **Layout structure** — Grid systems and responsive breakpoints
- **Motion principles** — Animation timing, easing, and transitions
- **Iconography** — Icon style, sizing, and usage guidelines

## Why Foundations Matter

Foundations provide the essential building blocks that enable teams to build consistent, scalable interfaces.

### Consistency

Foundations establish shared visual language across all products. When everyone uses the same color palette, typography scale, and spacing rules, interfaces feel cohesive and predictable.

### Predictability

Designers and developers can make decisions faster when the foundational rules are clear. No need to reinvent spacing values or guess at hover states — the answers are already defined.

### Accessibility

Foundations embed accessibility into the design system from the start. Color contrast ratios, focus states, and motion preferences are all considered at the foundation level.

### Scalability

As products grow, foundations ensure that new features integrate seamlessly with existing ones. The same design tokens and rules apply across all new work.

### Maintainability

When a foundational value needs to change — like updating a primary color — the change propagates through the entire system. This reduces technical debt and design inconsistencies.

## Relationship with Tokens

Foundations and design tokens work together to translate design decisions into implementation.

### Foundations Define Rules

Foundations are conceptual. They define the **rules and principles** that guide design decisions.

For example, the Color foundation defines:
- The purpose of semantic colors (primary, success, warning, error)
- Accessibility requirements for contrast
- How to use color to communicate meaning

### Tokens Implement Values

Design tokens are the **implementation layer**. They translate foundation rules into specific values that can be used in code.

For example, tokens implement the Color foundation:

| Foundation Concept | Token | Value |
|-------------------|-------|-------|
| Primary color | `color.primary.500` | `#3b82f6` |
| Success color | `color.success.500` | `#22c55e` |
| Text primary | `color.text.primary` | `#111827` |

### How They Work Together

1. **Foundations** define what we're trying to achieve (e.g., "Use primary color for main actions")
2. **Tokens** provide the specific values (e.g., `color.primary.500`)
3. **Components** use tokens to implement the foundation rules

This separation allows foundations to evolve conceptually while tokens provide stable implementation references.
